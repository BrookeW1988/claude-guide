Speaking Your Vision

You don't need to learn code. You need to learn how to delegate to code.

Think of Claude Code as a tireless, highly skilled engineer. Your job isn't to tell it how to build, but to be incredibly clear about what you want for your business. If you can describe it clearly, Claude can build it.

The Manual

The Brain: `claude.md`

This is a single text file that Claude reads every time it starts working for you. It holds the "rules of the house."

1. Technical Specs

What kind of engine are we using? (e.g., A mobile website or a data tracker).

2. Business Purpose

The "Why". Who is this for? What problem is it solving for your business?

3. Brand Voice & Rules

Your standards. "Use warm language" or "Stick to neutral brand colors."

Security

The Vault (.env): Protecting Your Keys

What is an API Key?

Think of an API Key as a Digital Keycard. It's a secret code that lets Claude "unlock" other appsâ€”like your Gmail, shop inventory, or a shipping service.

# Your secret .env file
GOOGLE_API_KEY=123-abc-456-xyz

Why use the `.env` file?

The .env file is The Vault. You put your digital keycards in here so Claude can use them, but no human eyes can see them when looking at the code. It keeps your business accounts safe.

!

Never paste keys in chat

Don't give Claude your actual API key directly in the message box. Ask it to "Help me save this key to my .env file" instead.

!

The Vault is Invisible

The .env file is usually hidden from public view automatically. It's designed to be your private digital safe.

Success Habits

Key Best Practices

ðŸ§ 

"Plan Mode" First

Before letting Claude write code, use Shift + Tab to enter Plan Mode. This lets you review the logic before any files are changed.

Review the plan as a CEO. If you don't like the design, change it before the build starts.
ðŸ’¾

Commit Often

Use Git to save working versions of your app. Think of it as hitting "Save As" on your work every time a feature works.

If Claude "breaks" a feature, you can instantly revert to the last working version.
ðŸš¦

Standard vs. YOLO

Standard Mode asks for your permission before every change. YOLO Mode runs automatically.

Beginner Founders should use Standard Mode to stay in control of the build.
ðŸ”‘

Secure Secrets

Always keep API keys in the .env file. Never type passwords directly into the chat box where they can be saved.

The .env file is your digital vault. It keeps your business credentials private.
